## 2024-05-15 - [Overly Permissive CORS Configuration]
**Vulnerability:** The application was using `@CrossOrigin(origins = "*")` on the `AnalyzeController`, allowing any website to make cross-origin requests to the API. This is overly permissive and can expose the application to unintended access.
**Learning:** Hardcoding wildcard origins (`*`) in the controller bypasses proper access controls and creates a significant security risk, especially for APIs that may handle sensitive data or actions.
**Prevention:** Instead of hardcoding CORS origins directly in the code, they should be configured via application properties (e.g., `app.cors.allowed-origins` in `application.properties`) and injected using Spring's property resolution (`@CrossOrigin(origins = "${app.cors.allowed-origins}")`). This allows different environments to have appropriate, restrictive configurations (like `http://localhost:5173` for local development and specific production domains for the live app).
