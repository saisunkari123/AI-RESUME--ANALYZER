## 2024-06-16 - CORS Misconfiguration Vulnerability
**Vulnerability:** The Spring Boot backend used an overly permissive CORS configuration (`@CrossOrigin(origins = "*")`) on the `AnalyzeController`. This allows any external website to make cross-origin requests to the API and read responses, which could lead to sensitive data exposure or Cross-Site Request Forgery (CSRF) style attacks (though primarily read-based in this case).
**Learning:** Developers often use wildcard CORS during early development for convenience, but it is dangerous in production because it undermines the same-origin policy, allowing malicious sites to access user data if authenticated or if sensitive info is returned.
**Prevention:** Always restrict CORS origins to only the trusted domains that need access. Use environment variables or configuration properties (e.g., `@CrossOrigin(origins = "${cors.allowed-origins}")`) to manage allowed origins dynamically per environment rather than hardcoding them, and never use `*` for production endpoints.
