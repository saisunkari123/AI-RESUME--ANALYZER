## 2024-06-30 - Fix Overly Permissive CORS Configuration and Stack Trace Leakage
**Vulnerability:** The Spring Boot backend was configured with `@CrossOrigin(origins = "*")`, which allows any domain to interact with the API, leading to potential Cross-Origin Resource Sharing (CORS) abuse. In addition, an exception block was using `e.printStackTrace()`, which logs raw stack traces to standard output, leading to potential sensitive information leakage and lack of proper logging context.
**Learning:** Hardcoded wildcard CORS headers combined with poor error handling can unintentionally expose backend structure and allow malicious sites to perform CSRF-like attacks or steal data. The use of `e.printStackTrace()` is discouraged because it lacks context, formatting, and standard logging levels, making monitoring and tracing difficult while potentially revealing internals.
**Prevention:** Always externalize CORS configuration (e.g., using properties like `app.cors.allowed-origins`) so it can be managed dynamically and restricted by environment. Ensure proper logging frameworks (like SLF4J) are used for error handling, avoiding stack trace leakage via direct stdout printing.
