## 2024-06-18 - Hardcoded Overly Permissive CORS Configuration
**Vulnerability:** The application was using an overly permissive wildcard CORS configuration (`@CrossOrigin(origins = "*")`) on the main `AnalyzeController`. This allows any domain to make cross-origin requests to the API, which can lead to data exposure and other cross-origin attacks when integrated with session-based authentication or internal networks.
**Learning:** Hardcoded wildcards in `@CrossOrigin` bypass the intended security restrictions of CORS. This pattern is often introduced during early development to "make things work" and forgotten later. It is specific to the Spring Boot REST endpoints in this codebase.
**Prevention:** Always use configurable properties for CORS allowed origins (e.g., `@CrossOrigin(origins = "${app.cors.allowed-origins}")`). Default the configuration to the specific trusted frontend origin (e.g., `http://localhost:5173`) and use environment variables to manage it across different environments.
