## 2024-06-14 - Overly Permissive CORS and Missing Input Validation
**Vulnerability:** The API endpoint allowed cross-origin requests from any domain (`@CrossOrigin(origins = "*")`) and lacked input validation for uploaded files and text length limits. The exception handling also printed full stack traces to stdout.
**Learning:** By default, Spring `@CrossOrigin("*")` is very convenient for initial development but leaves APIs exposed to CSRF-like attacks and unauthorized domain usage. Missing file type/size and length validation can easily lead to resource exhaustion (DoS) or unexpected processing errors when the downstream service (like PDFBox) receives malformed data.
**Prevention:** Always restrict CORS to explicitly known frontend origins (e.g., `http://localhost:5173`). Implement strict validation on `@RequestParam` inputs, particularly file uploads (check content-type and emptiness) and string lengths. Use secure logging rather than `e.printStackTrace()` to avoid exposing implementation details.
